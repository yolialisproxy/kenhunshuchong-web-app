tor browser

用tor翻墻得经历从慢到快的过程，快了后要持续快就不能再随便改动网桥的设置。找到了好的网桥，快起来后就轻易不要改动。改动后就又得慢上一阵子了，不知什么时候才快得起来，一切又得从头来始，从慢到快慢慢来……

最慢是第一步突破封锁，一旦改动了网桥的设置，就又得从头开始突破。

2021-7-30

tor browser

今天发现不只是改动网桥设置的问题，有时候改动设置反而能加快连接，这得看具体是什么情况。

  • 如果只是改动添加网桥里面的内容，而不改用不同的网桥方式，反而有可能使本来卡着的进度条快速突破。这也突破封锁最难的一步。具体的操作，其实也不需要真正改动里面的内容，只需把排在后面的某条改动到第一条，这样就行了。
  • 如果一定要改用不同的网桥方式，最好先把配置文件备份torrc。这文件先不要直接改动，直接改动会跟浏览器的自动修改互抢，抢来抢去里面的内容就容易出问题。如果一定要手动改动，也不是没办法，只要记得先把浏览器关闭就行。这样就不会再互相抢着修改里面的内容了。

2021-9-21

蓝灯

今天终于在linux上把蓝灯给搞定了，实现了浏览器之外的翻墙。

蓝灯装上后，最开始的速度还真是快得吓人，可惜高速免费听说只有500M/月。之后便会进入到正常速度的翻墙。

蓝灯的安装文件是deb文件，得先用debtap这命令转化，之后才能用pacman -U命令来安装。

安装后，双击它发现还不能正常运行。于是进入终端用命令运行来查看，发现提示说是有少了一个.so的库文件。

到/usr/lib下搜索同名文件，发现了一个只版本不同的高版本文件。用ln -s命令给那文件创建一个新的软链接，链接的文件名自然是那个少了的低版本文件名。接着一测试，蓝灯能正常运行了。

——上面用软链接改文件名的方法是在外文网站上学到的，不然还真没想到。

如果命令行工具也想访问蓝灯网络，需要为 Shell 设置两个环境变量 HTTP_PROXY 和 HTTPS_PROXY。

找到蓝灯在本地启动的具体端口号。打开蓝灯，依次选择 Settings -> ADVANCED SETTINGS 即可看到蓝灯在本地选择的端口号。

然后去 Shell 里执行以下两个命令设置环境变量：

export HTTP_PROXY=http://127.0.0.1:43385
export HTTPS_PROXY=http://127.0.0.1:43385

就完成配置了。

最后为了避免每次都要敲这么长的命令，写一个 Shell 函数：

# http proxy util
hp() {
  if [ "$1" = "enable" ]
  then
    PORT="43385"
    if [ -n "$2" ]
    then
      PORT="$2"
    fi

    export HTTP_PROXY=http://127.0.0.1:$PORT
    export HTTPS_PROXY=http://127.0.0.1:$PORT
  else
    export HTTP_PROXY=""
    export HTTPS_PROXY=""
  fi
}

把这都函数代码放到 Shell 启动脚本里。然后需要开启代理的时候执行 hp enable 即可，或是指定代理端口号 hp enable 33333。关闭代理执行 hp 即可。
