看bt磁力种子边下边看在手机上实现得很好，很想在电脑上实现。

网上一查，发现有一款软件叫btfs, 似乎很适合。

安装很简单，直接用yay -S btfs 就行。用起来有点麻烦，要用到两个命令，一个命令把bt种子文件挂载，另一个命令用来卸载，例如：

btfs test.torrent mnt

fusermount -u mnt

接下来是测试实际效果如何，结果试了一个又一个种子文件，都卡在了打开视频这一步。

最后不得不放弃了用这款，改为用webtorrent。

用过webtorrent后，选了个比较快的种子文件再来测试btfs，发现打开后看视频那速度真的很快，就是打开视频前确实有点慢。而一旦打开了，接下来就快了。

清理磁盘空间

测试过btfs后发现磁盘可用空间变小了，检查后发现用户目录下多了个btfs的目录，目录下多了很多文件，我猜应该是缓存文件。删除这些文件后，可用磁盘空间又恢复到原来的大小了。

对比qb、btfs、webtorrent、xunlei

这几款软件在linux平台上各有各的优势，但要说速度，可能还是迅雷的linux版最牛。

这几款软件快的时候都很快，但就算拿同一个种子来测试下载速度，实际上都还是会有很大的区别，因为各类软件都需要同类型用户在做种速度才能快得起来。这时候迅雷就有优势了，它一般不缺人做种，因而容易形成良性循环，导致同时做种的人越积越多。

同一个种子磁力链接，同样的网络环境，其他bt下载软件几天都找不到几个做种的，迅雷却很快就达到几M的下载速度，让人不得不服。不用迅雷还以为是废种，不知道为什么那么多废种，用了才知不是那么一回事。

2022-10-13

journal日志文件总是能轻易达到几个G，当系统空间不足时这无疑让系统雪上加霜，乱上加乱。这问题已经不容忽视，必须得处理。

网上一般都说可能用这个命令来解决：

# 仅保留500MB大小的日志文
$ sudo journalctl --vacuum-size=500M

我的经验是没这么简单，执行这命令时会提示需要特殊用户组用户才可以，事实就算加入特殊用户组再执行，结果也还是不如人意。

真能解决的办法是修改日志的配置文件:/etc/systemd/journald.conf。

SystemMaxUse=       [[全部日志最大占用磁盘空间]]

我半这设置为512M，重启系统后可以立马看到效果。我怀疑不用重启也可以，重启journal服务就行：systemd-journald。

journal log files can easily reach gigabytes, which can add to the mess when the system runs out of space. This problem cannot be ignored. It has to be dealt with.

The Internet generally says it’s possible to use this command:

# Keep only 500MB of logs
$ sudo journalctl --vacuum-size=500M

My experience is that it is not so simple. When executing this command, it will prompt you to use a special user group. In fact, even after joining a special user group, the result is still not satisfactory.

安装操作系统前一定要记得先给MBR做一个备份。

不管安装什么操作系统，操作系统的引导程序一般都会从修改MBR开始引导。

linux和windows使用的引导管理器不同，linux一般是用grub，windows一般是用NTLDR。开机后到底使用哪个引导管理器，决定权在MBR。只要在MBR被修改之前做了备份，就不怕装系统会对引导的管理造成什么意外。正常情况下装linux不怕，反正最后是由grub来引导，但就怕意外。最好还是不管安装什么操作系统都先给MBR做一个备份，把这当成一个好习惯来坚持。

这一次装虚拟机，大部分时候都感觉不到明显的卡，可能是因为这台电脑内存比原来多了4G。这使我对使用虚拟机开始感兴趣起来，希望能在大部分时候取代wine。

甚至我都想把原来的win10给删掉，空出来的空间用来在虚拟机上装一个win10，取代原来的。

甚至我都想不用win10了，直接在虚拟机上装win11。就不知到时候是不是会卡得厉害！

启动引导

这次装虚拟机和正常装机一样遇到了启动引导的麻烦，下了一个又一个ISO镜像换来换去都没用，最后还是借助了一个PE镜像里的引导修复工具来修复引导。

好在虚拟机上的系统要多加载一个PE镜像很是方便，加一个PE镜像就仿佛多加了一个虚拟光驱。

在当前虚拟机的‘设置–>系统–>主板–>启动顺序’那儿，把光驱的顺序调到前面，当启动这镜像系统时，就会像优先从光驱启动一样。不同是我们现在用的是虚拟系统上的虚拟光驱，而不是机械主板上用机械光驱。

虚拟磁盘

虚拟机上的系统空间不够用时，增加一块虚拟磁盘很简单，而且还可以动态分配大小。

我挂载的第一块虚拟磁盘是30G，分成两个区，系统区和软件区。很快便发现，软件区的空间用来做系统备份已经十分接近不够用了，看现在一个ghost系统备份就有4个G，而以后装了新软件后的剩余空间会更小。于是我又增加了一个10G虚拟磁盘专门用来做备份用，反正这磁盘上的空间不用时不占用磁盘空间。

远程桌面登录

rdesktop -f -r clipboard:PRIMARYCLIPBOARD -r disk:mydisk=/ 127.0.0.1

就这么一条命令，就登录了我的虚拟机win7，接下来就和平时使用win7没什么区别了。

再按一下win+alt+enter，虚拟机又回到了桌面模式，再按一次又可以再变回去。

像这样同时使用linux和windows，正是我期盼已久的模式。VirtualBox帮我实现了。

cpu消耗

大部分时候对cpu的消耗极低，但当开启病毒检查等程序时消耗自然会随之飚升，毕竟是两个系统同时在使用。

不足之处

最大的不足就是需要更多的磁盘空间来容纳一个又一个系统，让人忍不住想把各种系统装在虚拟机上，想用就用，方便快捷。

2021-9-19

买了台新机，便又多了个固态硬盘，想一机装两个固态硬盘。

按之前的经验，光驱位是识别不了固态硬盘的，所以之前那台东芝笔记本只有把固态硬盘装在主硬盘位置上，把机械硬盘换到光驱位。那样虽然能识别，但识别仍然困难：

  • 在linux下，要手动点击进入机械硬盘的文件夹，才真正激活 ;

  • 在windows下，需要休眠后再进入，才能够发现光驱位机械硬盘。

这次换了新机后，想试试新机的主板是不是更好，是不是能自动识别？结果新机识别的效果还不如旧机，在linux下效果一样，在windows下识别不了。

怎么办？

最后还是找到了解决的办法，但不是解决光驱位机械硬盘在windows下的识别问题，而是光驱位固态硬盘的识别。原来光驱位的电路板上有个小调节按钮，这按钮有三档，其中一档就是用来识别光驱位固态硬盘的。方法正确了，问题就简单，现在两个固态硬盘可以在一台笔记本上同时使用了，不论是在linux下还是windows下。

When I buy a new computer, I have a solid state drive. I want two solid state drives in one machine. As a rule of thumb, the SSD was not recognized by the optical drive, so the Toshiba had to install the SSD on the main drive and switch the mechanical hard drive to the optical drive. That would be recognisable, but still difficult: In Linux, you need to manually click into the folder of the mechanical hard drive to activate it. In Windows, you need to sleep and re-enter the system to discover the CD-ROM drive. After this new machine, want to try the motherboard of the new machine is not better, is it able to automatically identify? Results The new machine recognition effect is not as good as the old machine, the effect is the same under Linux, under Windows recognition is not. How to do? Finally, the solution was found, but it was not to solve the problem of recognition of optical drive mechanical hard disk in Windows, but the recognition of optical drive solid state hard disk. There is a small adjustment button on the circuit board of the original CD-ROM drive. This button has three gears, one of which is used to identify the CD-ROM SSD. Get it right, and the problem is simple: now you can use two solid-state drives on the same laptop, either Linux or Windows.
